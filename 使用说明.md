# 使用说明 — 51-MCU-Machine-Operation-Simulation

## 环境准备

### PC 端
- Windows 10+ 或 Linux
- CMake 3.16+
- C++20 编译器（MSVC 2022 / GCC 10+ / Clang 12+）
- MATLAB R2019b+（仅仿真层需要）

### MCU 模拟器（可选，无硬件调试用）
```powershell
pip install pyserial
```

### 真实 MCU 硬件（可选）
- STC89C52 或 STM32F103 最小系统板
- USB 转 TTL 模块（CH340 / CP2102）
- 8×8 LED 点阵 + 74HC595 + ULN2803 驱动电路

## 快速开始

### 第一步：MATLAB 仿真（生成轨迹数据）

在 MATLAB 中运行：

```matlab
% 可视化三种轨迹
run('matlab/day1_trajectory.m')

% 生成状态向量 → matlab/trajectory.txt
run('matlab/day2_state_generation.m')
```

![day1 效果：红=直线，绿=正弦，蓝=圆]
![day2 效果：离散状态点 + 速度箭头]

### 第二步：构建 C++ 项目

```powershell
cd 项目根目录
cmake -B build
cmake --build build --config Release
```

> **Visual Studio 用户**：直接双击 `TrajectorySystem.slnx` 打开即可，无需命令行。

### 第三步：运行（离线模式）

```powershell
.\build\Release\TrajectorySystem.exe
```

程序会：
1. 读取 `trajectory.txt`
2. 打印 MATLAB 状态向量
3. 运行 AI 路径规划
4. 生成轨迹点序列并输出到终端

## 串口联调（PC ↔ MCU）

### 方式一：MCU 模拟器（推荐先调试协议）

**终端 1** — 启动虚拟 MCU：
```powershell
python tests/mcu_simulator.py --port COM4 --baud 9600
```

**终端 2** — 运行主程序：
```powershell
# 需要有可用的串口对（如 com0com 虚拟串口）
$env:MCU_SERIAL_PORT = "COM3"
.\build\Release\TrajectorySystem.exe
```

模拟器会在终端绘制 8×8 LED 点阵，光点随轨迹移动：
```
┌────────┐
│········│
│········│
│····●···│  ← 当前位置
│········│
│········│
│········│
│········│
│········│
└────────┘
```

### 方式二：真实 MCU 硬件

1. 接线（参考 `firmware/hal_map.h` 引脚定义）
2. 烧录固件：

**51 MCU (SDCC)**：
```bash
sdcc -DMCU_8051 firmware/main.c firmware/protocol.c firmware/led_matrix.c firmware/queue.c
# 用 stcgal 或 STC-ISP 烧录生成的 .ihx
```

**STM32 (CubeIDE / arm-gcc)**：
```bash
arm-none-eabi-gcc -DMCU_STM32 -DSTM32F103xB -mcpu=cortex-m3 -mthumb \
  firmware/main.c firmware/protocol.c firmware/led_matrix.c firmware/queue.c \
  -T STM32F103C8Tx_FLASH.ld
```

3. 运行主程序：
```powershell
$env:MCU_SERIAL_PORT = "COM3"
.\build\Release\TrajectorySystem.exe
```

MCU 收到每帧后会回复 ACK（`0xCC` + 序号），控制层打印确认信息。

## DeepSeek AI 模式

设置 API Key 后，路径规划由 DeepSeek 大模型决策：

```powershell
$env:DEEPSEEK_API_KEY = "sk-xxxxxxxxxxxxxxxxxxxxxxxx"
.\build\Release\TrajectorySystem.exe
```

AI 会返回 JSON 格式的路径点和运动模式建议。如果 API 不可用（无 Key、超时、网络断开），自动降级为本地 A* + fakeAI，不影响程序运行。

## 移植到新 MCU

只需修改一个文件：`firmware/hal_map.h`

```c
// 1. 添加新的编译宏
#elif defined(MCU_STM32)
  #include "stm32f1xx_hal.h"
  // 2. 映射引脚
  #define LED_ROW_PORT   GPIOA
  #define LED_COL_PORT   GPIOB
  // 3. 实现 HAL_* 宏
  #define HAL_WRITE_ROW(val)  ...
  #define HAL_UART_RX_AVAIL() ...
```

编译时指定宏即可：
```bash
arm-none-eabi-gcc -DMCU_STM32 ...
```

## 常见问题

**Q: "无法打开 trajectory.txt"**
→ 先运行 MATLAB 的 `day2_state_generation.m`，或将 `matlab/trajectory.txt` 复制到 `.exe` 同目录。

**Q: CMake 构建失败 (WinHTTP 未找到)**
→ 确认使用 MSVC 编译器（`cmake -G "Visual Studio 17 2022"`）；Linux 下自动跳过 WinHTTP。

**Q: 串口无法打开**
→ 检查端口号是否被占用（关闭串口助手），确认设备管理器中 COM 号正确。

**Q: MCU 无反应**
→ 检查 TX/RX 是否交叉连接（PC TX → MCU RX，PC RX → MCU TX），波特率是否一致（默认 9600-8N1）。

**Q: LED 点阵不亮或闪烁**
→ 确认行选/列选方向（共阳/共阴），检查 `hal_map.h` 中 `HAL_WRITE_ROW`/`HAL_WRITE_COL` 逻辑电平。
